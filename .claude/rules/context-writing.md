# コンテキストファイル作成ルール

## 基本原則

- 1ファイル200行以内を目安にする（命令が増えると全体の遵守率が下がるため）
- 各行について「削除したら Claude が間違えるか」を問う。NOなら削除する

## 書くべき内容

- コードを読んでも分からないプロジェクト固有のコマンド・ワークフロー
- デフォルトと異なるコードスタイル
- 過去の失敗から得た知見

## 書いてはいけない内容

- コードから読み取れる情報の繰り返し
- 言語・フレームワークの標準的な慣習
- リンター・フォーマッターに委譲できるスタイルルール
- 自明な指示

## ファイル構成（3層構造）

| 層 | 場所 | 読み込みタイミング |
|---|---|---|
| Layer 1 | `CLAUDE.md` | 毎セッション |
| Layer 2 | `.claude/rules/` | パス一致時に自動読み込み |
| Layer 3 | `.claude/skills/` | オンデマンド |

- タスク固有の指示は Layer 1 ではなく `.claude/rules/` に置く
- 長い説明は別ファイルに切り出して `@` インポートで参照する

## アンチパターン

- 否定のみの記述（「〜するな」→「代わりに〜する」も書く）

## 運用方針

- 定期的に棚卸しして、不要な項目・Claude が指示なしで対応できる項目を削除する
