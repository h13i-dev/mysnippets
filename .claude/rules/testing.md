# Testing Rules

## ファイル命名規則

- `*.spec.ts` / `*.spec.tsx` → **ユニットテスト**（ユーティリティ関数・カスタムフック単体の検証）
- `*.test.ts` / `*.test.tsx` → **インテグレーションテスト**（複数モジュールを組み合わせた検証）

## 各テストの用途

- **VRT**: コンポーネントのスタイル変更時に実行。意図的変更は `npm run vrt:update` でスナップショット更新。
- **E2E**: ユーザーフロー全体の確認。開発サーバーを起動した状態で実行。
- **axe / nu**: コミット前に実行してアクセシビリティと HTML の妥当性を確認。

## カバレッジ方針

正常系・異常系・境界値のケースを網羅する。
